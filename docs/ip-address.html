myrules provide support to validate the IPv4 and IPv6.

Address Representation

IPv4 can be represented by four notations:

  • dot-decimal notation. Example: 192.0.2.235.
  • dot-binary notation. Example: 11000000.00000000.00000010.11101011.
  • dot-hexadecimal. Example: 0xC0.0x00.0x02.0xEB.
  • dot-octal notation. Example: 0300.0000.0002.0353.

The IPv6 are represented in 8 groups of 16 bits each. Each group is written as four hexadecimal digits and the groups are separated by colons (:). Example: 2001:0db8:85a3:03fa:0000:0000:0000:7344 or 2001:db8:85a3:3fa::7344.

The .mr-ip-address make validation of IPv4 (dot-decimal, dot-binary, dot-hexadecimal and dot-octal notations) and IPv6 (hexadecimal).

Define the accepted IP Representation

The .mr-ip-address class Specifies that the input value must be a valid IP address. It can be an IPv4 represented in dot-decimal notation, dot-binary notation, dot-hexadecimal notation and dot-octal notation and in IPv6 represented in hexadecimal digits.

You can restrict the type of IP Representation into your input field. Just add the .mr-ip-address class with the class that specifies the type of IP representation.

Examples: If you want to specify that the input value must be an IPv4 address, add the .mr-ip-address and .mr-ipv4 classes into the element class; To specify that the input value must be an IPv6 address use .mr-ip-address and .mr-ipv6; or if the input value must be an IPv4 in dot-decimal notation or an IPv6 add the .mr-ip-address, .mr-ipv4-dot-decimal and .mr-ipv6 classes into the element class.
