Customize myrules

By default, myrules validate an element when event change occurs (onchange) and when a form is submitted (onsubmit), but you can add the extra event, to validate the element when it gets user input (oninput), furthermore, you can add myrules colors style of validation status to the form elements and their feedbacks.

Validate element oninput

oninput event occurs immediately after the value of an element has changed, while onchange occurs when the element loses focus, after the content has been changed.

To validate an element oninput event, add a class .mr-validate-oninput into the class of <form> element.

Example: Validate element oninput event

The E-mail text fields is required.


Please fill out this text field!

Please input a valid e-mail!

it´s valid E-mail!

<form class="mr-validate-oninput">
<input type="email" class="mr" required placeholder="E-mail"> <br>
<div class="mr-feedback">
    <p class="mr-required-fb">Please fill out this text field!</p>
    <p class="mr-email-fb">Please input a valid e-mail!</p>
    <p class="mr-valid-fb">it´s valid E-mail!</p>
</div>

<input type="submit" value="submit">
<input type="reset" value="reset">
</form>

myrules colors style

Add the class .mr-colors-style into the class of <form> element to use myrules colors style.

Example: Adding myrules colors style on form element

The E-mail text fields is required.


Please fill out this text field!

Please input a valid e-mail!

it´s valid E-mail!

<form class="mr-colors-style">
    <input type="email" class="mr" required placeholder="E-mail"> <br>
    <div class="mr-feedback">
        <p class="mr-required-fb">Please fill out this text field!</p>
        <p class="mr-email-fb">Please input a valid e-mail!</p>
        <p class="mr-valid-fb">it´s valid E-mail!</p>
    </div>

    <input type="submit" value="submit">
    <input type="reset" value="reset">
</form>
