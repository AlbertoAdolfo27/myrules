Validation Attribute Validation Class Description Feedback Class
type="email" .mr-email Specifies that the input value must be a valid email .mr-email-fb

When validating an email, myrules uses the ValidityState.typeMismatch property of Constraint validation API, combined with a set of rules to define the valid syntax of an email address.

the typeMismatch property is used to check if the input value does not conform to the constraints set by the type='email' attribute of the element. In addition to checking the typeMismatch it is also checked if the input value matches the email address syntax defined by myrules.

myrules email address syntax

The format of an email address is local-part@domain

Local-part

  • The local-part of the email address may use any of these ASCII characters:
    • uppercase and lowercase Latin letters A to Z and a to z
    • digits 0 to 9
    • printable characters !#$%&'*+-/=?^_`{|}~
    • dot .
  • Dot cannot be the first or last character and it and cannot appear consecutively
  • You can change the printable characters

Domain

  • The domain consists of a list of dot-separated DNS labels, each label consists of these characters:
    • uppercase and lowercase Latin letters A to Z and a to z
    • digits 0 to 9
    • hyphen -
  • top-level domain names cannot all be numeric
  • Hyphen cannot be the first or last character

Changing printable characters for the local-part of the email

We've already seen the printable characters supported in the local-part of the email address, which are: !#$%&'*+-/=?^_`{|}~, you can change them by selecting some of them for the local-part of the email address.

For example, Gmail and outlook.com only allows creation of email addresses using alphanumerics, dot (.), underscore (_) and hyphen (-)

You can set the printable characters of your choice in the value of the data-printablechars or printablechars attribute value

you can add only printable characters supported in the data-printablechars or printablechars attribute (!#$%&'*+-/=?^_`{|}~).

Example: changing printable characters

  • The valid email address only allows alphanumerics, dot (.), underscore (_) and hyphen (-)
  • Try to submit an invalid email

<form>
    <!--Defining your printable characters in the data-printablechars attribute.-->
    <input type="email" class="mr" data-printablechars="_-" placeholder="Email"> <br>
    <!--Validation feedback-->
    <p class="mr-feedback mr-email-fb">Invalid email!</p>

    <input type="submit" value="submit">
    <input type="reset" value="reset">
</form>

Ingnoring myrules email address syntax

Example: ingnoring myrules email syntax

If you ignore myrules email syntax, my rules will only use the ValidityState.typeMismatch property to validate the input value. In this type of case, an email like "john.due@example" or "john.due@example1111.." might be valid, which by the default validation would be invalid.

The .mr-email-browser class will cause the myrules email address syntax to be ignored and only Constraint validation API's ValidityState.typeMismatch property to be considered

</form>
